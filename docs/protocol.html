MA Converter: A Universal Effect Size Translation Tool for Meta-Analysis — Protocol

Mahmood Alhusseini
Independent Researcher, Meta-Methods Group
Type: Protocol Protocol Reg. ID: MAC-2026-001 Date: 2026-03-31 License: CC BY 4.0 Open Access

Abstract

Background: Systematic reviewers frequently encounter heterogeneous effect measure reporting across trials — odds ratios (OR), risk ratios (RR), risk differences (RD), hazard ratios (HR), standardised mean differences (SMD), mean differences (MD), and correlation coefficients (r / Fisher's z). Converting between these measures requires exact distributional formulas, careful delta-method CI propagation, and clear accuracy warnings. No single freely available tool handles all nine conversions in a single interface.

Objectives: To build, document, and publish a universal effect size converter supporting OR ↔ RR ↔ RD ↔ NNT ↔ HR ↔ SMD ↔ MD ↔ Fisher's z with exact delta-method confidence interval propagation, a formula reference sheet, and batch-mode CSV import/export.

Methods: Single-file HTML application (no build step, no server dependencies) implementing Hasselblad-Hedges, Cox's log-HR approximation, exponential survival assumptions, and delta-method variance propagation. Batch mode supports up to 500 rows. Color-coded accuracy indicators signal approximation quality per conversion path.

Expected outcomes: Single and batch conversion, downloadable formula reference card, color-coded accuracy indicators (Exact / Good / Approximate / Rough), and round-trip verification fixtures.

1. Background & Rationale

Meta-analyses pool evidence from diverse trial reports. When primary studies report effect sizes on different scales, analysts must convert between them before pooling. The available conversions range from exact (Fisher's z ↔ r, which is a deterministic bijection) to rough approximations requiring baseline event-rate assumptions (OR ↔ HR under proportional hazards).

Manual conversion is error-prone: sign errors arise in log-scale transformations; delta-method formulas differ between OR and RR scales; and NNT requires a reference risk assumption that must be made explicit. A validated, auditable single-file tool eliminates these errors and serves as a transparent methodological artefact.

2. Objectives

  1. Implement all pairwise conversions among: OR, RR, RD, NNT, HR, SMD, MD, r, Fisher's z.
  2. Propagate 95% confidence intervals using the delta method throughout.
  3. Provide explicit accuracy grades (Exact / Good / Approximate / Rough) per conversion path.
  4. Support batch mode (CSV upload, up to 500 rows).
  5. Embed a formula reference panel covering all implemented formulas.
  6. Ship as a single self-contained HTML file suitable for offline use.

3.2 Core Conversion Formulas

Key formulas implemented in the engine:

OR → RR: RR = OR / (1 - p0 + p0*OR), where p0 = baseline risk RR → RD: RD = RR * p0 - p0 = p0*(RR - 1) OR → SMD (Hasselblad-Hedges): SMD = log(OR) * sqrt(3) / pi SE_SMD = SE_logOR * sqrt(3) / pi OR → HR (Cox approximation): HR ~ exp( log(OR) / (1 - p0*(1-p0)*log(OR)) ) Fisher's z ↔ r: z = atanh(r) = 0.5*ln((1+r)/(1-r)) r = tanh(z) SE_z = 1 / sqrt(n - 3)

3.3 Delta Method CI Propagation

For any monotone differentiable transformation g applied to a log-scale estimate theta with SE:

SE_g(theta) = |g'(theta)| * SE_theta CI_lower_g = g(theta - 1.96 * SE_theta) CI_upper_g = g(theta + 1.96 * SE_theta)

For OR ↔ RR conversion where both are ratio measures, the delta method is applied on the log scale to maintain positivity of the CI bounds.

3.5 Batch Mode

Users may upload a CSV file with columns: study, value, ci_lower, ci_upper, source_measure, target_measure. Optional columns: baseline_risk, n, sd. The engine processes up to 500 rows and returns a downloadable CSV with conversion results and accuracy grades.

3.6 Technical Implementation

The tool is implemented as a single self-contained HTML file (ma-converter.html, ~1,687 lines) with no external dependencies at runtime. The file embeds all CSS, JavaScript logic, and formula reference content. No server, build system, or internet connection is required for operation. The tool is published under CC BY 4.0 via GitHub Pages for open access.

4. Outcomes

  • Primary: Correct conversion of point estimate and 95% CI for all 9 source measures.
  • Secondary: Round-trip accuracy — OR → SMD → OR and Fisher's z → r → z return the original value within floating-point precision.
  • Usability: Single conversion mode operable in under 30 seconds; batch mode processes 500 rows in under 2 seconds.
  • Safety: Color-coded accuracy warnings visible on every conversion output.
  • Accessibility: All interactive elements keyboard-navigable; WCAG AA contrast on all text.

5. Safety Checks

The following safety checks are run before each release:

  1. Div balance: <div[\s>] count equals </div> count (excluding JS regex).
  2. Script tag integrity: no literal </script> inside template literals.
  3. Function name uniqueness across all JS modules.
  4. Element ID uniqueness across the full file.
  5. Round-trip fixtures: OR=2.0, CI=[1.2,3.3] → SMD → back within 1e-9.
  6. Fisher's z fixture: r=0.5 → z=0.5493 → r=0.5000 within 1e-10.
